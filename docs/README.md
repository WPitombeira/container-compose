# Documentation

This folder keeps long-form project information out of the root README while preserving a clear path for users, app integrators, and contributors.

## User Guides

- [Getting Started](GETTING_STARTED.md): installation, Compose discovery, common commands, stdin input, env files, and generated plans.
- [Compose Parity](COMPOSE_PARITY.md): how Container Compose follows Docker Compose, what is mapped, what is preserved with diagnostics, and what remains unsupported.

## Integration Guides

- [Integrations](INTEGRATIONS.md): high-level integration guidance for Container Desktop and other macOS tools.
- [Container Desktop Integration](CONTAINER_DESKTOP_INTEGRATION.md): detailed Swift package wiring, desktop snapshots, dry runs, execution adapters, runtime status, and stable handoff types.

## Project Guides

- [Architecture](ARCHITECTURE.md): package layers, loader strategy, planning model, and runtime boundary.
- [Roadmap](ROADMAP.md): prioritized workstreams for Compose parity, Apple Container verification, Container Desktop integration, and release readiness.
- [Development](DEVELOPMENT.md): local development commands, tests, smoke checks, and release hygiene.
- [Contributing](CONTRIBUTING.md): contribution expectations, pull request checklist, issue guidance, and coding standards.
